매년 바뀌는 입시제도 안에서 매 시즌 수백 명의 학생·학부모를 대면 컨설팅했습니다. 사전 설문 → 대면 상담 → 후속 관리 → 파이널 콜로 이어지는 고밀도 프로세스를 매 시즌 안정적으로 운영했고, 체계적이고 진정성 있는 컨설팅이라는 평이 학원 재등록·지인 추천의 리텐션으로 이어졌습니다.

이 환경에서 가장 깊이 배운 것은 수강생은 도구보다 먼저 문제 정의에서 막힌다는 감각입니다. 학부모가 묻는 질문 뒤의 진짜 불안, 학생이 말하지 않는 진짜 목표를 먼저 읽어야 전달이 꽂힙니다. 같은 내용도 상대의 배경·이해도·심리에 따라 설명의 순서와 호흡을 매번 다시 짜야 한다는 것을 수백 번 반복해서 확인했습니다.

대학 시절에는 6개월간 고등학생 대상 학원 집합 강의를 진행하며 1:1 상담·소집단 세션과는 또 다른 집합 포맷의 호흡을 별도로 검증했습니다.

무작정 솔루션을 꺼내기보다 "이 청중에게 지금 필요한 언어가 무엇인가"를 먼저 묻는 훈련 — 코드잇 공고의 "수강생이 어떤 것을 어려워하는지 잘 느끼고 공감"에 정확히 대응하는 4년의 근력입니다.

가르칠 자격은 "배운 사람"이 아니라 "직접 만들어 본 사람"에게서 생깁니다. AX · AI 활용 · AI 기초 세 레벨 모두에 바로 연결되는 경험을 갖고 있습니다.

02 Decisions

같은 기능을 구현하는 방법은 많지만,
어떤 판단은 실제 운영 환경에서만 나옵니다

세 프로덕트에서 내린 핵심 설계 판단 세 가지. 각각 왜 그렇게 결정했는가를 풀어 설명합니다 — 바로 이 지점이 교육 현장에서 가장 부족한 자료이기도 합니다. "이 기능은 이렇게 만듭니다"는 쉽게 찾을 수 있지만, "이 상황에서는 왜 이쪽이 아니라 저쪽을 선택해야 하는가"는 만들어 본 사람만이 가르칠 수 있습니다.

01

왜 최신 Realtime 대신 2초 폴링을 백업으로 남겼나?

Context

현장은 Wi-Fi 품질이 일정하지 않습니다. WebSocket이 수 초 끊기면 라운드 오픈·투표 마감 같은 타이밍 크리티컬한 순간에 UI가 stale해지고, 이는 현장 운영 자체의 붕괴로 직결됩니다.

Decision

Realtime을 Primary, 2초 간격의 얇은 폴링을 Fallback으로 병행. 네트워크가 깜빡여도 최대 2초 안에 상태가 복구됩니다.

Trade-off

서버 부하 미세 증가(Edge Functions 무료 티어 내 수렴) ↔ 화면 stale 위험 제거.

02

왜 완전 자동화가 아니라 Human-in-the-loop를 선택했나?

Context

전 공정 자동 생성은 구현이 쉽지만, YouTube 등 플랫폼이 완전 자동 생성 콘텐츠 제재를 점점 강화하는 방향으로 움직이고 있었습니다. 운영 계정·채널 보호가 우선 조건이었습니다.

Decision

짧은 편집 개입 구간을 의도적으로 남김. 스크립트 수정·팩트 보정·최종 승인을 사람이 통과시키는 HITL 게이트 유지.

Trade-off

자동화 효율 일부 감소 ↔ 리스크를 플랫폼이 아니라 사람과 나눠 가짐.

03

왜 Electron 앱에서 API를 직접 호출하지 않았나?

Context

데스크톱 앱은 코드가 클라이언트에 통째로 배포됩니다. API 키를 번들에 심으면 asar 언패킹·메모리 덤프만으로 평문 키가 그대로 노출됩니다.

Decision

클라이언트에는 사용자 JWT만 두고, 실제 API 호출은 Supabase Edge Functions 프록시가 대신 수행. 전형적인 BFF(Backend for Frontend) 패턴.

Trade-off

추가 레이턴시 수십 ms ↔ 키 노출 방지 + 서버 측 레이트 리밋·쿼터·로그 통합 제어.

코드잇 지원 서류 [2]번 문항에 대한 답변을 포트폴리오 위에 그대로 펼쳐놓았습니다. 모두 실제로 만들어 운영한 프로덕트와 직접 실험해 본 설계 판단을 기반으로 한 주제들이며, 코드잇의 AX · AI 활용 · AI 기초 세 레벨에 직접 대응합니다.

AI 기초 · MODULE 01

생성형 AI 이해와 실무 입문

Target AI를 처음 배우는 비전공자 · 일반 실무자 · 입문 수강생
Format 90분 특강 · 3시간 입문 강의 · 1일 기초 워크숍 · 온라인 녹화 강의

생성형 AI가 무엇이고, 어디까지 할 수 있으며, 어디서 한계가 생기는지를 헷갈리지 않게 쪼개서 설명합니다. 프롬프트 몇 개를 외우는 방식이 아니라, "어떤 문제에 어떤 도구를 왜 쓰는가"를 기초부터 잡아 주는 과정. 대치동 4년간 비전문가에게 복잡한 제도를 풀어 설명해 온 전달 감각을 그대로 적용합니다.

AI 활용 · MODULE 02

업무 생산성 · 자동화 실습

Target 기획 · 마케팅 · 운영 · 디자인 등 비개발 실무자
Format 3시간 실습 강의 · 1일 집중 워크숍 · 4~6주 실습형 과정 · 부트캠프 모듈

문서 정리, 리서치, 초안 작성, 반복 업무 자동화까지 — 내일부터 바로 적용 가능한 활용법을 실습 중심으로 다룹니다. 참가자 자신의 실제 업무를 예제로 사용해, "강의실에서 배운 것"이 아니라 "업무에서 써먹는 것"이 남도록 설계합니다. Claude · ChatGPT · Gemini 역할 분담 원칙도 함께 다룹니다.

AX · MODULE 03

조직 도입과 사람·도구 역할 설계

Target 팀 리드 · 중간관리자 · HRD 담당 · 사내 AX 담당자
Format 2시간 세미나 · 반일 워크숍 · 사내 교육 프로그램

AX는 도구 소개가 아니라 업무 재설계의 문제라는 전제에서 출발합니다. "무엇을 AI에게 맡기고, 무엇을 사람이 판단해야 하는가"를, 실제 프로덕트에 의도적으로 남겨 둔 Human-in-the-loop 사례를 기반으로 설명합니다. 영상 자동화 앱에 별도 팩트 체크(Perplexity) 단계를 넣은 판단도 여기에 포함됩니다.

커리큘럼 설계 · MODULE 04

실습형 커리큘럼 · 초급자 온보딩 설계

Target 교육팀 · 부트캠프 운영진 · 사내 교육 담당자
Format 커리큘럼 기획 세션 · 파일럿 강의 설계 · 과정 리디자인

초급자가 어디서 막히는지, 어떤 예제가 이해를 돕는지, 어떤 순서로 개념과 실습을 배치해야 이탈하지 않는지를 기준으로 실습형 커리큘럼과 교안 구조를 설계합니다. 대치동 4년간 "사전 설문 → 대면 상담 → 후속 관리 → 파이널 콜"로 운영한 구조화된 전달 프로세스를 교육 콘텐츠 제작에 그대로 적용합니다.

위 4개 모듈은 코드잇 AX · AI 활용 · AI 기초 기업 교육에 바로 투입 가능한 구조입니다. 온라인 녹화 강의, B2C 부트캠프, 기업 맞춤 프로그램 포맷 전환도 동일한 설계 원칙으로 진행할 수 있습니다.

    "무엇을 자동화할 것인가"가 아니라 "어느 지점에 사람 판단을 남길 것인가"를 먼저 설계해 온 관점으로, 도입 ROI와 운영 판단 기준을 정리. 크롤링 앱 설계 시 API 비용 증가를 감수하고 리서처 기회비용 관점에서 ROI를 결정한 실제 의사결정 사례 교재화 가능.

설명의 정확성을 지키는 방식

강의에서 가장 중요한 것은 전달력만이 아니라 정확성입니다. 모호한 내용을 그럴듯하게 넘기지 않고, 공식 문서 · 실사용 검증 · 예제 사전 실행을 기준으로 설명을 고정합니다.

  • 공식 문서 우선 · 모델 사양·API 변경은 공식 자료에서 확인 후 설명
  • 예제 사전 실행 · 강의용 코드는 모두 실행 검증 후 사용
  • 다중 모델 교차 검증 · 모호한 내용은 복수 LLM으로 교차 확인

Architecture & Challenges

행사 특성상 하나의 이벤트에서 50~100명이 동시에 같은 RPC를 호출하는 순간이 있습니다. 라운드 오픈·투표 마감·매칭 결과 공개 — 초 단위 타이밍에 서버가 견뎌야 하고, 네트워크가 깜빡여도 화면이 stale 상태로 멈추지 않아야 했습니다. 일부 시나리오에서 Supabase Realtime의 안정성이 행사 운영 요구치에 미치지 못해, 핵심 화면(매칭 결과 공개·라이브 운영 화면 등)은 2초 폴링을 1차 채널로 두고 Realtime을 보조로 두는 hybrid 구조로 전환했습니다. 재접속 시에는 SHA-256 해시 기반 resume_token으로 세션을 복구하고, 동시 RPC 호출에는 ON CONFLICT 멱등 처리와 advisory lock을 조합해 race condition을 방어합니다.

Operations-First Engineering

  • 관리자 기능별 ON/OFF 제어 — 행사 도중 특정 기능만 즉시 끌 수 있어야 함. 어드민 패널 RPC 레벨 토글.
  • bcrypt 기반 어드민 인증 + 레이트 리밋 — 10분 내 10회 실패 시 자동 락, 공격 로그 1시간 자동 정리.
  • D+1 운영 반영 사이클 — 행사 현장 피드백을 다음 행사 전까지 닫는 운영 모델. 빠른 릴리즈 그 자체가 아니라, 피드백 → 분석 → 반영 → 검증 루프의 길이를 짧게 유지하는 데 초점.

-- 참가자 등록 · 재접속 시 idempotent 동작
-- resume_token 평문은 저장하지 않고 SHA-256 해시만 보관
create or replace function public.register_participant(
  p_room_code     text,
  p_user_id       uuid,
  p_display_name  text,
  p_resume_token  text
) returns jsonb
language plpgsql security definer as $$
declare
  v_participant_id uuid;
  v_token_hash     text;
begin
  v_token_hash := encode(digest(p_resume_token, 'sha256'), 'hex');

  insert into participants (room_code, user_id, display_name, resume_token_hash)
  values (p_room_code, p_user_id, p_display_name, v_token_hash)
  on conflict (room_code, user_id) do update
    set display_name      = excluded.display_name,
        resume_token_hash = excluded.resume_token_hash
  returning id into v_participant_id;

  return jsonb_build_object(
    'participant_id', v_participant_id,
    'status',         'ok'
  );
end; $$;

TypeScript 
// 단계별로 강점이 다른 LLM에 라우팅 — hallucination 최소화
type Stage = 'analyze' | 'draft' | 'structure' | 'verify';

const ROUTE: Record<Stage, LLMProvider> = {
  analyze:   'gemini',      // 긴 영상 입력 · 멀티모달 분석
  draft:     'claude',      // 한국어 문장 톤
  structure: 'gpt',         // JSON 스키마 안정성
  verify:    'perplexity',  // 최신 팩트 체크
};

export async function runStage<T>(
  stage:  Stage,
  input:  StageInput
): Promise<T> {
  const llm    = clients[ROUTE[stage]];
  const result = await llm.invoke(input, {
    timeout: STAGE_TIMEOUT[stage],
    retries: 2,
  });
  return validate(stage, result);
}

Why a Supabase Edge Functions Proxy?

데스크톱 앱은 코드가 클라이언트에 통째로 배포되기 때문에, API 키를 앱 내부에 심으면 리버스 엔지니어링으로 탈취 가능합니다. Edge Functions를 프록시로 두면 사용자는 인증 토큰만 받고, 실제 Apify·LLM·YouTube API 호출은 서버에서 대신 수행합니다. 전형적인 BFF(Backend for Frontend) 패턴을 데스크톱 앱에 적용한 구조입니다.

Why Apify?

Instagram·TikTok은 공식 API가 극히 제한적이거나 상업용 크롤링을 금지합니다. Apify는 관리형 크롤링 서비스로 구현·운영 복잡도를 낮추고, apify~instagram-post-scraper와 clockworks~free-tiktok-scraper 같은 actor들을 바로 조합할 수 있습니다. free-tier actor의 가용성 리스크는 paid actor로의 fallback 경로를 미리 설정해 흡수합니다.

TypeScript · Deno 
// 클라이언트 JWT만 검증 · API 키는 서버 env에만 존재
const APIFY_TOKEN = Deno.env.get('APIFY_TOKEN')!;

Deno.serve(async (req) => {
  // 1. 유저 JWT 검증
  const jwt = req.headers.get('Authorization');
  const { data: { user } } = await supabase.auth.getUser(jwt);
  if (!user) return new Response('Unauthorized', { status: 401 });

  // 2. 레이트 리밋 · 쿼터 제어
  if (await isRateLimited(user.id)) {
    return new Response('Too Many Requests', { status: 429 });
  }

  // 3. 프록시 호출 — 키는 서버에서만 주입
  const { endpoint, payload } = await req.json();
  const res = await fetch(`https://api.apify.com/v2/${endpoint}`, {
    method:  'POST',
    headers: {
      'Authorization': `Bearer ${APIFY_TOKEN}`,
      'Content-Type':  'application/json',
    },
    body:    JSON.stringify(payload),
  });
  return new Response(await res.text(), { status: res.status });
});
